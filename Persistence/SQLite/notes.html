ACID Properties and Transactions

1.What are Transactions?

A transaction is a sequence of database operations executed as a single unit. Transactions ensure data integrity and consistency by following the ACID properties.

2.What are ACID Properties?

  • Atomicity: Ensures that a transaction is all-or-nothing. If one part fails, the entire transaction is rolled back.
  • Consistency: Ensures the database remains in a valid state before and after a transaction.
  • Isolation: Ensures transactions do not interfere with each other.
  • Durability: Ensures committed transactions persist even in the event of a system crash.

3.Is a System Without Transactions Useful?

A system without transactions can still function but lacks guarantees for consistency and reliability. It may be acceptable for simple applications like logging but is not suitable for financial systems.

Properties of a File System

  • Atomicity at the file level
  • Consistency via journaling (e.g., ext4, NTFS)
  • Isolation through access control
  • Durability through persistent storage

4.Effects of Missing 'A' from ACID Properties

Missing Atomicity

If atomicity is missing, partial transactions may be written, causing inconsistencies.

When is it okay? In logging systems or real-time analytics where missing some records is acceptable.

5.Effects of Missing 'C','I' and 'D' from ACID Properties

Missing Consistency

If consistency is missing, the database may contain invalid data.

When is it okay? In NoSQL databases with eventual consistency, like Cassandra.

Missing Isolation

If isolation is missing, transactions may interfere with each other, causing dirty reads or lost updates.

When is it okay? In high-performance analytics or real-time leaderboards.

Missing Durability

If durability is missing, committed transactions may be lost after a crash.

When is it okay? In caching systems like Redis where data loss is acceptable.

Choosing a Database

  • (k, v) Databases (e.g., Redis): Best for key-value lookups.
  • Relational Databases: Best for structured data and transactions.
    • SQLite: Best for single-application usage.
    • PostgreSQL: Best for distributed and multi-application usage.
