Project Troubleshooting Notes

Problems Faced & Solutions

1. Pip Install --editable . Failed

Issue: The installation failed due to a Python version mismatch in pyproject.toml.

Solution:

Updated pyproject.toml to use compatible Python constraints:

[tool.poetry.dependencies]
python = ">=3.8"

Then ran:

pip install --editable .
Prompt: "pip install --editable . error: metadata-generation-failed"

2. Command Not Found (sriya-wf-dynamic-streaming)

Issue: The installed command was not found in the system PATH.

Solution:

Checked if the command was installed:

which sriya-wf-dynamic-streaming

Added it to the PATH if needed:

export PATH=$HOME/.local/bin:$PATH
Prompt: "sriya-wf-dynamic-streaming not found"

3. ImportError: cannot import name 'processing_functions'

Issue: The processing_functions.py file was in the wrong directory.

Solution:

Moved the file to the correct location:

mv processing_functions.py sriya_wf_dynamic_streaming/

Updated the import in commands.py:

from sriya_wf_dynamic_streaming.processing_functions import upper_case
Prompt: "ImportError: cannot import name 'processing_functions'"

4. Unexpected Extra Argument Error

Issue: The CLI was not correctly structured to recognize commands.

Solution:

Fixed commands.py by explicitly defining the command:

@app.command("process")

Reinstalled the package:

pip install --editable .
Prompt: "Got unexpected extra argument (input.txt)"

Final Fix

After applying these fixes, sriya-wf-dynamic-streaming process input.txt worked successfully.
